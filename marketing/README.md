# Marketing & Outreach Pack — Role-Indexed Attention

Goal: maximise visibility of the paper, get the work into hands of researchers
at frontier AI labs (Anthropic, OpenAI, Google DeepMind, Meta AI, Mistral,
Cohere, etc.), and translate that visibility into a research / engineering
position.

This folder contains:

| File | Use |
|------|-----|
| `tweet_thread.md` | Twitter / X thread, 12 posts, day-of-arXiv announcement |
| `linkedin_post.md` | Professional version, longer-form, for LinkedIn / professional networks |
| `hn_post.md` | Hacker News submission text + first comment to seed discussion |
| `cold_email.md` | Templates for emailing researchers at frontier labs |
| `recruiter_pitch.md` | One-pager for recruiters / hiring managers |
| `personal_landing.md` | What to put on your personal website / GitHub profile |
| `qa_prep.md` | Common questions and prepared answers |

## Recommended sequence

1. **Day 0**: arXiv preprint goes live + Zenodo deposit + GitHub repo public.
2. **Day 0**: Tweet thread, LinkedIn post.
3. **Day 1**: Hacker News post (Tuesday or Wednesday morning US time gets the
   most traction).
4. **Day 2-7**: Cold-email researchers at frontier labs (one per day, not bulk).
5. **Day 7+**: Talk requests, podcast outreach.

## Honest framing rule

Throughout: never claim things we haven't measured. The paper is genuinely
strong on its merits; over-claiming would erode trust quickly. Concrete numbers
+ reproducible code + honest limitations is the most persuasive combination.
